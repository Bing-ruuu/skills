# /to-md — 文本/代码/PDF 转 Markdown

将输入的文字、代码或 PDF 转换为格式良好的 Markdown，并保存到当前目录的 `.md` 文件。

## 工作流程

### 1. 解析输入

判断输入是**内联内容**还是**文件路径**：

- **文件路径**（以 `.py` / `.js` / `.ts` / `.md` / `.json` / `.txt` 等结尾，或包含 `/` 或 `\`）：
  - 使用 `Read` 工具读取文件内容
  - 根据扩展名推断语言类型
- **PDF 文件**（以 `.pdf` 结尾）：
  - 使用 `Read` 工具读取 PDF 内容（支持多页）
  - 提取文本和结构信息
- **内联内容**（直接粘贴的文本或代码）：
  - 视为纯文本，尝试检测是否像代码
  - 如有明显代码特征（缩进、括号、关键字等），标记为代码块

### 2. 内容处理

**文件类型判断**：
- `.pdf` → PDF 文件，进入 PDF 处理流程
- `.py` / `.js` / `.ts` / `.md` / `.json` / `.txt` 等 → 代码/文本文件
- 其他或内联内容 → 根据内容特征判断

**PDF 处理逻辑**：
- 使用 `Read` 工具读取 PDF，提取所有页面的文本内容
- 保留 PDF 中的标题层级（h1/h2/h3）
- 将表格转换为 Markdown 表格格式
- 图片暂时用 `![image](placeholder)` 占位，标注原位置
- 代码块保持原有缩进和格式

**代码检测逻辑**：
- 文件路径 → 用扩展名映射语言（`.py` → python，`.js` → javascript 等）
- 内联内容 → 检测缩进（4空格或Tab）、行尾分号、关键字（function/def/class/import）等特征
- 无法确定时，默认为纯文本

**Markdown 组装**：
```
# [文件名或自动生成标题]

[内容]

---
*Generated by /to-md on [日期]*
```

- 如果是代码片段，用 \`\`\`language 代码块包裹
- 如果是纯文本，直接写入
- PDF 内容保留原有结构和层级

### 3. 输出文件

- **文件名**：
  - 文件路径：`[original-name].md`
  - PDF 文件：`[original-name].md`
  - 内联内容：`converted-[timestamp].md`
- **保存位置**：当前工作目录
- **操作**：使用 `Write` 工具写入文件

### 4. 反馈

告知用户：
- 检测到的内容类型（代码/纯文本/PDF，语言类型）
- 生成的文件名
- 文件路径

## 边界情况

| 情况 | 处理 |
|-----|-----|
| 空输入 | 提示用户输入内容或文件路径 |
| 文件不存在 | 报错：文件不存在，请检查路径 |
| 文件太大（>1MB） | 提示文件过大，手动分段处理 |
| 路径含特殊字符 | 用引号包裹路径后读取 |
| PDF 加密/损坏 | 报错：无法读取 PDF，请检查文件是否完整 |
| PDF 扫描件（无文本层） | 提示：此 PDF 为扫描件，无法提取文本 |

## 示例

**输入（文件路径）**：
```
/to-md C:\Users\test\main.py
```
**输出**：
```
[已转换] main.py → main.md
语言：python
保存至：C:\Users\test\main.md
```

**输入（PDF 文件）**：
```
/to-md C:\Users\test\document.pdf
```
**输出**：
```
[已转换] document.pdf → document.md
类型：PDF
页数：12
保存至：C:\Users\test\document.md
```

**输入（内联代码）**：
```
/to-md
def hello():
    print("world")
```
**输出**：
```
[已转换] 代码片段 → converted-20260407.md
语言：python
保存至：C:\Users\当前目录\converted-20260407.md
```
